CPU Live Lab: LoRA Training with distilgpt2 & RAG with nomic-embed-text

Two models, two jobs

This lab uses two different models because training and retrieval are different tasks. You do not need a GPU.

Model Used in Job
distilgpt2 train_model.py Learn LoRA fine-tuning on CPU — update a tiny adapter so the model mimics your Q&A style
nomic-embed-text rag.py Turn text into vectors for semantic search — find the right facts before asking Ollama
llama3.2 rag.py, Modelfile High-quality chat answers via Ollama — weights are not changed by our CPU training script

Why distilgpt2 for training?

  • Small (~82M parameters) — fine-tunes on a laptop CPU in about a minute.
  • Same learning goal — you still see loss go down, LoRA adapters save to disk, and you run inference with peft.
  • Llama 3.2 is too large for CPU training — a 3B model would take hours or days per run and needs far more RAM.
  • Not an Ollama model — the adapter is Hugging Face format (adapter_model.safetensors), not a GGUF file for ollama create.

Think of it this way

distilgpt2 is a training bicycle — small, safe, fast to learn on. llama3.2 is the production car you drive via Ollama for real chat quality. You learn mechanics on the bicycle; you deliver answers with the car (often via RAG).

Why nomic-embed-text for RAG?

  • Embeddings ≠ chat — RAG needs to search your documents by meaning, not generate sentences. Embedding models output number vectors; chat models output text.
  • Runs in Ollama — ollama pull nomic-embed-text gives you a small, fast embedding model on CPU.
  • Semantic match — "bread recipe" can match "How to bake bread" even when words differ.
  • No retraining — add new rows to data.json and RAG picks them up without running train_model.py again.

Path A — LoRA training (distilgpt2)

LoRA trains a small adapter on top of the base model instead of updating all weights.

Step 1: Prepare data
Edit data.json with instruction / output pairs. Preview with python load_data.py.
Step 2: Train on CPU
Run python train_model.py. LoRA updates ~0.7% of parameters. Output goes to lora_adapter/.
Step 3: Test in Python
Load the base model + adapter with peft and generate text (see script below).

Load & test your trained adapter

from pathlib import Path
import torch
from peft import PeftModel
from transformers import AutoModelForCausalLM, AutoTokenizer

ADAPTER_DIR = Path("lora_adapter")
BASE = "distilgpt2"

tokenizer = AutoTokenizer.from_pretrained(ADAPTER_DIR)
base = AutoModelForCausalLM.from_pretrained(BASE, dtype=torch.float32)
model = PeftModel.from_pretrained(base, ADAPTER_DIR)
model.eval()

question = "What is XYZ ORG?"
prompt = f"### Instruction:\n{question}\n\n### Response:\n"
inputs = tokenizer(prompt, return_tensors="pt")

with torch.no_grad():
    out = model.generate(**inputs, max_new_tokens=80, pad_token_id=tokenizer.eos_token_id)

print(tokenizer.decode(out[0], skip_special_tokens=True))

No Modelfile ADAPTER line for distilgpt2

Ollama's ADAPTER command expects a GGUF LoRA trained on the same base (e.g. llama3.2). Your lora_adapter/ folder is Hugging Face format for distilgpt2 — use Python to run it, not ollama create.

Path B — RAG (nomic-embed-text + llama3.2)

Use this when you want accurate answers from your documents in Ollama without retraining.

Step 1: Embed documents
nomic-embed-text converts each Q&A in data.json into a vector.
Step 2: Retrieve on question
Embed the user's question and find the closest chunks by cosine similarity.
Step 3: Chat with context
Send retrieved text + question to llama3.2 via Ollama's /api/chat.

Commands

ollama pull llama3.2
ollama pull nomic-embed-text

python rag.py

Path C — Ollama persona only (Modelfile)

Which path when?

Goal Use
Learn how LoRA / training loops work train_model.py + distilgpt2
Answer from company docs / FAQs rag.py + nomic-embed-text + llama3.2
Fixed personality or behavior rules Modelfile SYSTEM prompt

Core takeaway

After retraining, re-run python train_model.py to refresh lora_adapter/. For RAG, update data.json and re-run python rag.py. For Ollama personas, edit the Modelfile and run ollama create again.
