Masterclass: Local LLM Deployment using Ollama & Custom Modelfiles

1. Introduction to Ollama

What is Ollama? Ollama is an open-source framework designed to bundle, manage, and run Large Language Models (LLMs) locally on your own machine. It abstracts away the complex math environments and driver configurations, acting essentially like "Docker for AI Models".

Why run models locally?

  • Data Privacy: No data leaves your machine or local server environment.
  • Zero Cost: No API token costs or subscription fees for generation.
  • Offline Capability: Run inference entirely without an active internet connection.

2. Essential CLI Commands

To interact with Ollama, you will primarily use the terminal. Below are the basic commands every programmer needs to memorize:

A. Starting the Ollama Service

Initiates the local background server engine that exposes the inference API ports.

ollama serve

B. Downloading a Model

Downloads weights from the official Ollama registry without instantly launching the prompt environment.

ollama pull llama3.2

C. Running a Model Interactively

This command downloads the model (if not already present), loads it into RAM/VRAM, and launches a terminal chat interface.

ollama run llama3.2

Tip: To exit an active chat session, type /exit or press Ctrl + D.

D. Managing Local Models

Check what models are available locally, see details, or remove models to clear disk space:

# List all downloaded models
ollama list

# Show architecture and parameters of a specific model
ollama show llama3.2

# Delete a model from storage
ollama rm llama3.2

3. Customizing Models with a Modelfile

A Modelfile is a configuration file used to build custom models with specific personalities, bounds, and parameters. It is highly analogous to a Dockerfile.

Key Modelfile Instructions:

  • FROM (Required): Specifies the foundational base model.
  • PARAMETER: Configures hyperparameters (e.g., temperature, context size).
  • SYSTEM: Sets the core instructions or system prompt persona.

Practical Classroom Project: Creating 'PinkCity Bot'

Create a file literally named Modelfile with no extension, and paste the following blueprint:

# Step 1: Choose the base engine
FROM llama3.2

# Step 2: Set the creativity level (Temperature)
# Lower values (e.g., 0.2) = highly factual and repetitive.
# Higher values (e.g., 0.9) = highly creative and unpredictable.
PARAMETER temperature 0.3

# Step 3: Write the System Persona Architecture
SYSTEM """
You are 'PinkCity Bot', an expert AI tour guide developed for XYZ ORG Jaipur.
You must answer questions exclusively about Rajasthan's history, local heritage sites like Hawa Mahal, Amer Fort, and Chokhi Dhani.
Rules:
1. Always begin your response by greeting the user with 'Khamma Ghani!'.
2. Be incredibly respectful and polite.
3. End your answers by recommending a local delicacy like Dal Baati Churma or Pyaaz Kachori if contextually appropriate.
"""

Building and Executing Your Custom Model:

Run these commands inside the directory containing your Modelfile:

# 1. Compile the Modelfile into a brand new local model
ollama create pinkcity-bot -f ./Modelfile

# 2. Run your newly created custom model
ollama run pinkcity-bot

4. Understanding "Training" vs. "Customization"

Important Distinction for Engineers: Ollama is an Inference Engine, not a training engine. It does not calculate gradients or adjust internal mathematical weights of base layers directly.

How Customization Flows in the Industry:

When engineering real-world LLM systems, customization is divided into distinct pathways depending on the business requirements:

A. System Prompting (What we did with Modelfile)

We wrap instructions around the model's environment. The model does not learn new facts; it is simply instructed to behave in a specific style and filter its pre-existing knowledge pool.

B. Fine-Tuning / Training (The External Pipeline)

To teach a model completely new technical skills or deep domain proprietary data, engineers perform Fine-Tuning outside of Ollama:

  1. Compute frameworks like PyTorch, Unsloth, Axolotl, or Hugging Face are run on heavy cluster GPUs.
  2. The dataset trains structural parameters, outputting lightweight LoRA Adapters (usually .bin or .safetensors files).
  3. We then import that trained file into Ollama using the ADAPTER directive in the Modelfile to run it cleanly.
# Importing a trained LoRA adapter file into Ollama
FROM llama3.2
ADAPTER ./path/to/my-trained-lora-weights.bin
SYSTEM "You are an assistant trained on specialized medical documentation files."
